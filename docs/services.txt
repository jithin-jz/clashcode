# ==============================================================================
# CLASHCODE - PROJECT ARCHITECTURE AND FEATURES DOCUMENTATION
# ==============================================================================

This document provides an exhaustive technical explanation of the services, 
applications, and logic that power the CLASHCODE platform.

--------------------------------------------------------------------------------
1. SYSTEM ARCHITECTURE
--------------------------------------------------------------------------------

CLASHCODE uses a microservices architecture orchestrated via Docker:

- API GATEWAY (Nginx): Central entry point (Port 80). Routes traffic to 
  appropriate services and serves static/media files.
- CORE SERVICE (Django/DRF): The primary business logic engine. Manages 
  users, challenges, social features, and economy.
- AI SERVICE (FastAPI): Handles RAG-based AI assistance, hints, and code 
  analysis using LangChain and ChromaDB.
- CHAT SERVICE (FastAPI): Real-time communication using WebSockets, 
  Redis Pub/Sub, and hybrid SQL/DynamoDB storage.
- DATABASES: PostgreSQL (Relational data), Redis (Caching/PubSub/OTP), 
  ChromaDB (Vector data).

--------------------------------------------------------------------------------
2. CORE SERVICE (DJANGO) - APP-BY-APP DEEP DIVE
--------------------------------------------------------------------------------

The Core service is a modular Django project located in `services/core`. Each 
app handles a specific domain of the platform.

A. AUTHENTICATION (`auth`)
   - JWT Implementation: Uses a dual-path authentication system. It accepts 
     tokens via the `Authorization: Bearer` header (for API clients) AND 
     `HttpOnly` cookies (for secure browser-based sessions).
   - OAuth Lifecycle: Supports GitHub and Google. Logic includes email-based 
     account bridging—if an OAuth email matches an existing account, it 
     automatically links the profiles instead of creating a duplicate.
   - OTP Security: Passwordless login via Email OTP. Uses Redis-backed 
     throttling to prevent brute-force attacks (5 attempts max per window).

B. ADMINISTRATION (`administration`)
   - Audit Logging: Every administrative action (blocking users, deleting 
     posts, global notifications) is recorded in the `AdminAuditLog` model 
     with actor IP, User-Agent, and a unique `X-Request-ID` for traceability.
   - Analytics Engine: Aggregates system-wide metrics (User growth, XP 
     inflation, store revenue, and challenge completion rates) to provide a 
     high-level platform health dashboard.

C. CHALLENGES & LEARNING (`challenges` & `learning`)
   - Level Map: A structured campaign of coding tasks with order-based 
     progression. It supports "Global" levels for everyone and 
     "Personalized" levels (generated for specific users).
   - Star System: Performance is graded (0-3 stars) based on completion 
     speed relative to a `target_time_seconds` defined per challenge.
   - Submission Engine: Client-side Pyodide validation runs challenge tests,
     and the `learning` app processes completion, stars, and XP rewards.
     It also serves as the gateway to the AI service for hints and analysis.

D. XP & ECONOMY (`xpoint`)
   - Transaction Safety: Uses PostgreSQL `select_for_update` within atomic 
     transactions to ensure XP balances are updated consistently, 
     preventing race conditions during simultaneous rewards or purchases.
   - Reward Cycle: A 7-day cyclical streak system. Users earn varying 
     amounts of XP for daily check-ins, resetting every week.

E. STORE & COSMETICS (`store`)
   - Cosmetic Categories: Allows users to spend XP on Themes, Fonts, 
     UI Effects, and Victory Animations.
   - Inventory Management: Purchases are one-time per user. The system 
     stores "Equipped" state in the UserProfile meta-data.

F. PAYMENTS (`payments`)
   - Razorpay Integration: Handles the Order-Verify flow. It generates 
     a server-side signature which must be verified against the 
     client-side Razorpay response before crediting XP.
   - Tiered Packages: Manages fixed INR-to-XP conversion rates.

G. NOTIFICATIONS (`notifications`)
   - Generic Signaling: Uses Django's `ContentType` system and `GenericForeignKey` 
     to allow notifications to point to any object (Posts, Challenges, etc.).
   - FCM Integration: Signals automate the creation of notifications 
     and trigger Firebase Cloud Messaging (FCM) pushes for real-time mobile/web engagement.

H. SOCIAL & MEDIA (`posts` & `users`)
   - Social Graph: Implements a follower/following system with optimized 
     ManyToMany relationships.
   - Media Pipeline: Handles multi-part image uploads for social posts, 
     storing them in a structured `/media/posts/` hierarchy.

--------------------------------------------------------------------------------
3. AI SERVICE (FASTAPI) - RAG & HINTS
--------------------------------------------------------------------------------

The AI service (`services/ai`) provides intelligent assistance:

- RAG Architecture: Leverages LangChain to combine challenge descriptions 
  (Context) with user code to provide specific, non-generic help.
- Vector Storage: Uses ChromaDB with HuggingFace embeddings to store 
  challenge metadata for efficient semantic retrieval.
- Tiered Hinting: Provides 3 levels of hints (ranging from conceptual clues 
  to logic snippets). Hints are cached to ensure consistency and 
  logical progression.

--------------------------------------------------------------------------------
4. CHAT SERVICE (FASTAPI) - REAL-TIME
--------------------------------------------------------------------------------

The Chat service (`services/chat`) manages high-concurrency messaging:

- WebSockets & Pub/Sub: Uses FastAPI WebSockets and Redis Pub/Sub to 
  broadcast messages across multiple service instances.
- Hybrid Storage: Stores recent messages in SQL for quick retrieval, with 
  long-term archival potential in DynamoDB (via `dynamo.py` implementation).
- State Tracking: Manages user "Online/Offline" presence and "Typing..." 
  indicators via Redis.

--------------------------------------------------------------------------------
5. INFRASTRUCTURE & SECURITY
--------------------------------------------------------------------------------

- Reverse Proxy: Nginx handles SSL termination (production), rate 
  limiting, and prevents direct access to application servers.
- Environment Management: Strict `.env` separation for sensitive keys 
  (Razorpay Secret, Django Secret, AI API Keys).
- Containerization: Every service is horizontally scalable and 
  isolated via Docker Compose.
