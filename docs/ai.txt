# AI Service: Zero to Hero Deep Dive

Welcome to the internal deep dive of the coc project's AI architecture. This guide will take you from a basic understanding to an expert level on how we leverage LLMs and RAG to provide intelligent coding assistance.

## High-Level Architecture

The AI service is a standalone component that interacts with the Core Service, ChromaDB, and LLM Providers (like Groq).

[Architecture Diagram Summary]
- User requests a hint or code analysis via the Frontend.
- Frontend calls Core Service (Django).
- Core Service proxies the request to the AI Service (FastAPI) after permission and balance checks.
- AI Service fetches challenge context from Core and search context from ChromaDB.
- AI Service calls the LLM (Groq/Llama-3.3) for inference.
- The hint or analysis is returned back through Core to the Frontend.

---

## Core Components

### 1. FastAPI Gateway (services/ai/main.py)
The AI service is built with FastAPI. Its primary job is handling hint requests, code analysis, and coordinating data flow.

- Endpoints:
    - `POST /hints`: Generates tiered hints for coding challenges.
    - `POST /analyze`: Performs detailed code review and analysis.
    - `GET /health`: Basic health check.
- Security: Uses an `X-Internal-API-Key` header to verify requests from the Core service.
- Workflow:
    1. Receives `user_code`, `challenge_slug`, and `hint_level` (for hints).
    2. Fetches challenge data (description, starter code, tests) from Core.
    3. Performs RAG to find similar challenge patterns.
    4. Orchestrates LLM call via LangChain.
    5. Sanitizes output to prevent answer leakage.

### 2. LLM Factory (services/ai/llm_factory.py)
Uses a Factory pattern for LLM provider management.
- Primary Provider: Groq (Llama-3.3-70b-versatile).
- LLM Engine: LangChain's `ChatOpenAI` wrapper pointing to Groq's OpenAI-compatible endpoint.
- Fallback logic: Currently configured to retry with the primary provider.

### 3. Prompt Engineering (services/ai/prompts.py)
Tiered hint strategy and expert review persona:
- System Prompt: Expert Coding Tutor persona with strict safety rules.
- Hint Levels (Implementation):
    - Level 1 (Gentle): A nudge or question.
    - Level 2 (Moderate): A more direct clue.
    - Level 3 (Significant): Explains the concept and implementation strategy.
- Code Review: Provides Findings, Edge Cases, Complexity, and Refactor Suggestions.

### 4. Output Sanitization
To maintain educational integrity, the service includes a `sanitize_guidance_output` utility that:
- Removes Markdown code blocks.
- Strips lines that look like code implementation (def, class, if/else, etc.).
- Ensures only conceptual guidance reaches the user.

---

## Retrieval-Augmented Generation (RAG)

1. Embedding: `HuggingFaceEmbeddings` (all-MiniLM-L6-v2) for local vectorization.
2. Vector Store: Connects to a standalone ChromaDB server via `chromadb.HttpClient`.
3. Search: Retrieves the 2 most similar challenges (excluding the current one) to provide context for the LLM.
4. Context Injection: Injected as `rag_context` to help the LLM understand curriculum style and previous patterns.

---

## Interaction with Core & XP

Core Service manages the economy:
1. XP Cost: Hints cost XP (10 for level 1, 20 for level 2, etc.).
2. Permission Check: Verified in `challenges/views.py`.
3. Proxy: Core acts as a secure gateway, injecting the internal API key.

## Setup & Deployment

- Docker: Containerized non-root Slim Python image.
- Environment: Managed via `.env` (passed via Docker Compose).
- Logging: Structured logging with custom formatting (`logger_config.py`).

## Code Quality & Documentation

The project adheres to high standards of maintainability and clarity:
- **Code Formatting**: The entire Python codebase is formatted using `black` to ensure consistent style and readability.
- **API Documentation**: Comprehensive OpenAPI (Swagger) documentation is generated via `drf-spectacular`. This includes:
    - Detailed request/response schemas for all endpoints.
    - Explicit HTTP status codes (200, 201, 204, 400, etc.).
    - Inline serializers for complex and computed data structures.
- **Monitoring**: Built-in health checks and Celery task status views are fully documented and integrated for production reliability.

## Summary: Zero to Hero

The AI service is a sophisticated pipeline of context fetching, vector search, tiered prompting, and secure proxying. It serves as the "Brain" of the platform, providing safe, educational guidance without spoiling solutions.
