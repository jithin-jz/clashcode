CLASHCODE - ARCHITECTURE
========================

The System:
-----------
CLASHCODE is a distributed app. Everything runs on Amazon EKS (Kubernetes) in production.

Simplified Flow:
----------------
[ User Browser ]
       |
       v
[ AWS Load Balancer ]
       |
       +--- [ Core API (Django) ] ----> [ PostgreSQL & Redis ]
       |
       +--- [ Chat Service (FastAPI) ] -> [ DynamoDB ]
       |
       +--- [ AI Service (FastAPI) ] --> [ Pinecone ]
       |
       +--- [ Executor (FastAPI) ] ----> [ Docker Sandboxes ]

Service Roles:
--------------
- Core: The "brain". Handles users, login, and the game state.
- Chat: Handles real-time messages. Uses DynamoDB for long-term storage.
- AI: Uses "Retrieval-Augmented Generation" (RAG) to give smart hints.
- Executor: A safe room where we run user code. It blocks dangerous stuff.

Security:
---------
- We use JWT tokens that rotate often.
- User code is locked in a Docker container with no internet.
- AWS IAM roles ensure services only touch what they need.

Deployment:
-----------
We use Terraform for infrastructure and GitHub Actions for the CI/CD pipeline.
